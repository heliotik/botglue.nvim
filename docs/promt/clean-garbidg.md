**Задача:** Разработать план превращения одиночного файла `prompt-optimizer.lua` в полноценный Neovim-плагин с правильной структурой, документацией и инфраструктурой качества.

**Текущее состояние:**
- Один файл `botglue.nvim/prompt-optimizer.lua` (~425 строк) содержит всю логику
- Промты захардкожены в таблице `PROMPTS` (строка 13)
- Отсутствует стандартная структура плагина, тесты, линтеры

**Требования к результату:**

1. **Структура плагина по шаблону** — взять за основу https://github.com/ellisonleao/nvim-plugin-template:
   - `lua/botglue/init.lua` — точка входа
   - `lua/botglue/*.lua` — модули (prompts, ui, claude, config)
   - `plugin/botglue.lua` — автозагрузка команд

2. **Конфигурируемость** — пользователь должен переопределять промты через `setup()`:
   ```lua
   require('botglue').setup({
     prompts = { optimize = "...", explain = "..." }
   })
   ```

3. **Инфраструктура качества** — по образцу https://github.com/heliotik/300/blob/master/Makefile:
   - Линтер: luacheck или selene
   - Форматирование: stylua
   - Тесты: plenary.nvim или busted
   - CI: GitHub Actions

4. **Документация:**
   - `README.md` на английском: установка, использование, конфигурация
   - Vimdoc в `doc/botglue.txt`

5. **Публикация** — репозиторий готов к установке через lazy.nvim/packer

**Формат ответа:** Дизайн-план с этапами, структурой файлов и зависимостями между задачами.
